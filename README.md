# Система авторизации

Современная форма авторизации на React, разработанная в соответствии с макетами и требованиями к API. Система поддерживает двухэтапный вход: по номеру телефона и одноразовому коду (OTP), а также сохранение сессии пользователя.

## Стек технологий

- **React** 
- **TypeScript**
- **React Hook Form**
- **Axios**
- **CSS3**

## Основные возможности

1.  **Двухэтапная авторизация**:
    -   Ввод номера телефона и получение OTP-кода.
    -   Ввод 6-значного проверочного кода.
2.  **Умная валидация**: Полное соответствие таблице требований (запрет ввода букв, проверка длины кода, обязательные поля).
3.  **Управление сессией**: 
    -   Сохранение JWT-токена в `localStorage`.
    -   Автоматическое восстановление данных пользователя при обновлении страницы (`/api/users/session`).
4.  **Таймер повторной отправки**: Динамический отсчет времени до возможности запросить код снова.
5.  **Обработка состояний**: Индикация загрузки (Loading states) и вывод ошибок от API.

## Реализовано 
```text
POST /api/auth/otp — Запрос кода на номер телефона.
POST /api/users/signin — Проверка кода и получение токена.
GET /api/users/session — Получение данных текущего пользователя по токену.
```

1.   Клонированеи репозитория
```bash

cd authorization
```
2.   Устновка зависимостей
```bash
npm install
```
3.   Запуск
```bash
npm run dev
```
